# Contributing to iris_classification

Thank you for your interest in contributing to iris_classification! We appreciate your help in making this project better.

## How Can I Contribute?

There are many ways to contribute, including:

- Reporting bugs or issues
- Suggesting new features or improvements
- Submitting code via pull requests
- Improving documentation
- Helping with testing and reviewing

## Getting Started

1. Fork the repository and clone it locally.
2. Create a new branch for your feature or bug fix.
3. Follow the coding style and guidelines (see below).
4. Write or update tests for new code or features.
5. Run all tests locally to ensure nothing breaks.
6. Submit a pull request with a clear description of your changes and reference any related issues.

## Code Style and Guidelines

- Follow existing code style and formatting.
- Use PEP8 style for Python code where applicable.
- Write clear, concise commit messages.
- Document new functions and modules.
- Ensure that any changes preserve pipeline reproducibility.

## Reporting Issues

If you encounter bugs or want to request features, please open an issue on GitHub with detailed information and reproducible examples where possible.

## Code of Conduct

By contributing, you agree to abide by our [Code of Conduct](https://github.com/derrickmichaelj/dsci-522-group-project/blob/main/CODE_OF_CONDUCT.md), which outlines expected behavior within this community.

## Need Help?

If you have any questions, feel free to open an issue for assistance.

Thank you for helping make iris_classification great!
